Error: Profiling Rules Mismatch

There was an issue with the profiling rules you provided. Profiling rules must be in the domain of the data, meaning:

  • The predicates (conditions) in the rules should correspond to the column names in your data.
  • Please make sure that the column names mentioned in the rules exist in the uploaded data.

Example:

# If the data contains columns 'age', 'income', and 'location', the profiling rule should reference these columns directly.
# For example:
# - 'age > 30'
# - 'income < 50000'
# - 'location == "New York"'

If you continue to experience issues, please double-check the rules and try again.

You can go back and upload the correct files or refer to the documentation for more information.
